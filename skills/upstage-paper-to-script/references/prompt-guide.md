# Paper-to-Script Prompt Guide

## 발표 스크립트 생성 가이드

### 스타일별 특성

| 스타일 | 청중 | 어투 | 깊이 | 시간 배분 |
|--------|------|------|------|-----------|
| formal | 학회 참석자 | -습니다 | 높음 | 도입 짧게, 방법론 상세 |
| seminar | 연구실/학과 | -요/-습니다 | 중간 | 배경 설명 충분히 |
| lab-meeting | 연구실 동료 | 편한 어투 | 핵심만 | 디스커션 시간 여유 |

### 시간 배분 가이드 (15분 기준)

| 섹션 | formal | seminar | lab-meeting |
|------|--------|---------|-------------|
| 도입/배경 | 2분 | 3분 | 2분 |
| 관련 연구 | 2분 | 2분 | 1분 |
| 방법론 | 4분 | 4분 | 4분 |
| 실험/결과 | 4분 | 3분 | 4분 |
| 결론 | 2분 | 2분 | 2분 |
| Q&A 안내 | 1분 | 1분 | 2분 |

### 2단계 처리 (긴 논문)

논문 텍스트가 10,000자를 초과하면 자동으로 2단계 처리:

**Stage 1 — 구조적 요약:**
- 문제 정의 (Problem)
- 방법론 (Method)
- 실험 결과 (Results) — 수치 그대로 유지
- 핵심 Figure/Table 설명
- 한계 및 향후 연구

**Stage 2 — 스크립트 생성:**
- 요약을 바탕으로 발표 스크립트 생성
- 핵심 수치와 모델명이 보존됨

이전 12,000자 하드 트렁케이션 대비 정보 손실이 크게 줄어듭니다.

### 청중 맞춤 (`--audience-context`)

청중 특성에 따라 스크립트가 자동 조절됩니다:

| 청중 | 조절 사항 |
|------|-----------|
| "비전공자 포함" | 전문 용어에 부연 설명 추가, 비유 활용 |
| "NLP 전문가" | 기술적 디테일 유지, 관련 연구 비교 강조 |
| "학부생 대상" | 기초 개념 설명 포함, 단계적 설명 |
| "산업계 참석자" | 실용적 적용 사례 강조, 성능 수치 중심 |

```bash
python3 run_paper_to_script.py \
  --text-file paper.txt --duration 15 --style seminar \
  --audience-context "비전공자 포함, CS 전공이 아닌 공학부 학생"
```

### 핵심 Figure 강조 (`--key-figures`)

특정 Figure/Table을 스크립트에서 중점적으로 다루도록 지시합니다:

```bash
python3 run_paper_to_script.py \
  --text-file paper.txt --duration 20 --style formal \
  --key-figures "Figure 2,Table 1,Figure 5"
```

- 지정된 Figure/Table은 해당 섹션에서 더 상세하게 설명
- 슬라이드 cue에 Figure 전환 표시 포함

### 좋은 발표 스크립트의 요소

1. **훅(Hook)**: 첫 30초에 청중의 관심을 끄는 질문이나 사실
2. **내러티브**: 문제 → 시도 → 해결의 스토리라인
3. **핵심 수치**: 구체적인 성능/결과 숫자 반드시 언급
4. **전환**: 섹션 간 자연스러운 연결 멘트
5. **마무리**: take-away message 명확히

### 입력 텍스트 최적화

- 10,000자 이하: 원문 직접 사용 (최적)
- 10,000자 초과: 자동 2단계 요약 처리
- 표/그래프 설명은 텍스트로 풀어서 제공하면 더 좋은 결과
