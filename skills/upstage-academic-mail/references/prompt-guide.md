# Academic Mail Prompt Guide

## 한국어 학술 이메일 작성 가이드

### 수신자별 톤 매트릭스

| 수신자 | 호칭 | 문체 | 예시 |
|--------|------|------|------|
| 교수님 | OO교수님 | -습니다/-겠습니다 | "검토해 주시면 감사하겠습니다" |
| 동료 | OO님/이름 | -요/-습니다 | "확인해 주시면 감사합니다" |
| 외부 | OO님/직함 | -습니다 (비즈니스) | "회신 부탁드리겠습니다" |

### 이메일 구조

1. **인사**: 시의적절한 인사 (계절/시간대 반영)
2. **자기소개**: 첫 연락 시 소속/이름 (재연락 시 생략)
3. **본론**: bullet points → 자연스러운 문단
4. **요청/마무리**: 명확한 action item
5. **서명**: 이름 + 소속

### 답장 모드 (`--reply-to`)

이전 이메일 파일을 제공하면 답장 모드로 동작합니다:

- 이전 이메일의 톤과 형식을 분석하여 일관된 답장 작성
- 이전 이메일에서 언급된 주제를 자연스럽게 참조
- "지난 메일에서 말씀하신 ~에 대해" 식의 연결 표현 사용
- 자기소개 생략 (이미 아는 사이이므로)

```bash
# 교수님 이메일에 답장
python3 run_academic_mail.py \
  --bullets "목요일 오후 가능, 발표자료 첨부 예정" \
  --recipient-type professor --tone formal \
  --reply-to professor_email.txt
```

### 맥락 주입 (`--context`)

추가 맥락을 제공하면 이메일 도입부에 자연스럽게 녹여 넣습니다:

- 만남 경위: "지난 EMNLP에서 뵌 OO대학교 OO입니다"
- 공유 배경: "OO 교수님 소개로 연락드립니다"
- 시간적 맥락: "3월 미팅에서 논의한 내용의 후속입니다"

### CC 안내 (`--cc-note`)

CC 수신자가 있을 때 왜 포함했는지 설명하는 문구를 추가합니다:
- "공동지도교수님께도 진행 상황 공유 드립니다"
- "팀원들에게도 참고 차 함께 보냅니다"

### 제목 작성 규칙

❌ 금지: "문의드립니다", "안녕하세요", "요청 건"
✅ 권장: 핵심 키워드 + 구체적 내용

| 나쁜 예 | 좋은 예 |
|---------|---------|
| 문의드립니다 | [미팅 요청] 졸업논문 3장 피드백 관련 |
| 안녕하세요 | [데이터셋 공유 요청] EMNLP 발표 후속 |
| 요청 건 | [일정 확인] 4/15 중간발표 시간 조율 |

### 상황별 공손함 자동 조절

| 상황 | 특징 | 자동 감지 단서 |
|------|------|----------------|
| 첫 연락 | 자기소개 포함, 최대 공손 | --reply-to 없음, --context에 "처음" |
| 후속 연락 | 간결한 연결, 중간 공손 | --reply-to 있음 |
| 리마인더 | 정중하되 명확 | "확인", "회신", "마감" 키워드 |
| 긴급 | 긴급 표현 + 사과 | "급히", "오늘 내", "마감 임박" |

### 프롬프트 최적화 팁

- bullet point는 구체적일수록 좋음 (날짜, 숫자 포함)
- `--context`로 배경 정보 추가 시 더 자연스러운 이메일 생성
- `--reply-to`와 `--context`를 함께 사용하면 가장 맥락이 풍부한 답장 가능
- formal + professor 조합이 가장 높은 경어 수준
- casual + colleague 조합이 가장 편한 톤

- Solar 모델은 한국어 존칭 체계를 잘 이해
