# n8n으로 만든 Upstage 기반 논문 요약 자동화: Hugging Face Paper를 한국어로 구조화 정리하기

연구하면서 가장 자주 깨지는 루틴이 있다.

1. 논문은 계속 쌓이고
2. 읽다가 중간에 끊기고
3. 나중에 보면 "이 논문 왜 저장했지?" 상태가 된다.

이번엔 이 문제를 줄이려고 **n8n + Upstage**로 작은 자동화 서비스를 만들었다.
입력은 Hugging Face paper URL, 출력은 한국어 구조화 요약(JSON)이다.

---

## 만들고 싶었던 것

- 논문 URL 몇 개를 던지면
- 각 논문을 동일한 템플릿으로 요약하고
- 바로 내 리서치 액션으로 이어지는 형태

요약 포맷은 아래처럼 고정했다.

- 한 줄 요약
- 문제 정의
- 핵심 방법
- 결과
- 한계
- 다음 액션 아이템

핵심은 "잘 요약"보다 "비교 가능하게 요약"이다.

---

## 왜 Upstage를 썼나

이 자동화에서 Upstage를 선택한 이유는 3가지였다.

1. **한국어 결과 품질**
   - 영어 논문을 한국어로 일관되게 설명해주는 게 중요했다.
2. **구조화 출력(JSON) 강제**
   - downstream 자동화(Discord/Notion 적재)를 위해 파싱 가능성이 핵심.
3. **실무형 프롬프트 내성**
   - 과장 금지, 수치 보존 같은 지시를 넣었을 때 결과가 안정적이었다.

---

## 아키텍처

워크플로우는 단순하다.

1. **Manual Trigger**
   - `urls[]`와 `targetLanguage`를 입력받는다.
2. **URL 펼치기(Code)**
   - URL 배열을 item 단위로 분해한다.
3. **본문 추출(HTTP Request)**
   - `r.jina.ai` 리더를 사용해 페이지를 텍스트로 변환한다.
4. **Upstage 요약(HTTP Request)**
   - Solar 모델에 구조화 요약을 요청한다.
5. **JSON 파싱(Code)**
   - 모델 응답을 객체로 변환한다.
6. **결과 묶기 + 응답**
   - 최종적으로 `count + summaries[]` 형태로 반환한다.

---

## n8n에서 특히 유용했던 포인트

- 코드 노드 2개만으로도 배열 처리/후처리가 깔끔했다.
- Webhook 기반이라 외부 서비스(앱, 슬랙봇, 디스코드봇)와 쉽게 연결된다.
- 추후 확장(요약 저장, 알림 발송, 주간 digest)이 빠르다.

---

## 보안: API 키를 깃허브에 올리지 않는 법

실수로 키가 올라가면 자동화보다 사고 수습이 더 커진다.

내가 적용한 최소 수칙:

1. 실제 키는 `.env`에만 저장
2. 저장소에는 `.env.example`만 커밋
3. `.gitignore`에 `.env` 추가
4. 블로그/캡처/코드 예시에는 마스킹 값만 사용
5. 유출 시 즉시 폐기(revoke) 후 재발급

---

## 실제 사용 루틴

- 아침: 흥미로운 논문 URL 3~5개 수집
- 점심 전: 자동 요약 일괄 실행
- 저녁: `action_items` 기준으로 실험 후보 1~2개 선정

이렇게 하면 "읽을 거리"가 "실행 항목"으로 바뀐다.

## 마무리

요약 자동화의 목표는 시간을 아끼는 게 아니라,
**읽은 것을 의사결정 가능한 형태로 바꾸는 것**이다.

n8n과 Upstage 조합은 이 목적에 꽤 잘 맞는다.
작게 시작해서, 본인 리서치 루틴에 맞춰 확장하면 체감이 크다.
